# Customer Support Persona Evaluator

An LLM-powered evaluation framework that measures how different AI agent personas perform on real customer support scenarios — and proves that prompt design has a quantifiable impact on response quality.

---

## The Problem

When building AI support agents, choosing a persona (professional, empathetic, urgent) is usually a subjective decision. This project makes it objective: which persona actually produces better responses, and by how much?

---

## How It Works

```
Generate complaints → Run 3 personas → Grade each response → Compare results
```

**Step 1 — Dataset generation**
10 realistic customer complaints are generated by Claude (I used haiku) across 5 categories: late delivery, wrong item, refund request, damaged item, and account issues. Complaints vary in tone — some calm, some frustrated — to stress-test each persona.

**Step 2 — Persona responses**
Each complaint is passed to Claude 3 times, once per persona, each with a distinct system prompt:
- **Professional** — formal, precise, solution-focused
- **Empathetic** — warm, validating, human
- **Urgent** — direct, fast, action-first

**Step 3 — LLM-as-grader evaluation**
A separate Claude instance acts as a quality analyst, scoring each response across 3 dimensions:
- **Empathy** — does the response acknowledge the customer's feelings appropriately?
- **Accuracy** — does it actually address the complaint?
- **Resolution** — does it provide a clear, actionable next step?
(I used Claude Sonet)

**Step 4 — Report generation**
Scores are averaged across all test cases and compared per persona.

---

## Prompt Engineering Techniques Used

- **claude-haiku-4-5** — dataset generation (fast, cost-efficient for structured tasks)
- **claude-sonnet-4-5** — response grading (stronger reasoning for nuanced evaluation)

**Persona design via system prompts** — each persona is controlled entirely through the system prompt. Tone, phrasing style, priority (speed vs warmth vs precision) are all defined there with no changes to the user-facing prompt.

**LLM-as-judge** — instead of hand-labelling outputs, a grader model evaluates responses against consistent scoring criteria. This scales evaluation without manual effort.

**JSON prefill trick** — Claude is prompted with `{"` as a prefill to force valid JSON output without markdown wrapping, eliminating parse errors reliably.

**Structured output enforcement** — the grader prompt specifies exact JSON field names and types, with a response example, ensuring consistent machine-readable scoring every run.

## Results

```
(haiku result:)
=======================================================
         PERSONA EVALUATION REPORT
=======================================================
Persona          Empathy  Accuracy  Resolution   Overall
-------------------------------------------------------
Professional         7.6       8.7         7.7       8.0
Empathetic           9.0       8.9         8.3       8.7
Urgent               7.1       8.8         8.9       8.3
=======================================================

Best overall persona: EMPATHETIC
Score: 8.7/10

Key insight:
  Empathetic outperformed Professional by 0.7 points overall.
  This shows prompt persona design has measurable impact on response quality.


(sonnet result:)
=======================================================
       PERSONA EVALUATION REPORT
=======================================================
Persona            Empathy  Accuracy  Resolution   Overall
-------------------------------------------------------
Professional           7.7       9.2         8.3       8.4
Empathetic             8.9       9.0         8.5       8.8
Urgent                 4.5       8.3         8.6       7.1
=======================================================

Best overall persona: EMPATHETIC
Score: 8.8/10

Key insight:
  Empathetic outperformed urgent by 1.7 points overall.
  This shows prompt persona design has measurable impact on response quality.

```

**What the numbers show:**
Sonnet penalized the Urgent persona's empathy score significantly
(7.1 → 4.5) compared to Haiku. Sonnet applies stricter standards
for emotional intelligence in support contexts — Haiku was more
lenient. This shows that grader model selection is not a neutral
choice: it encodes different quality standards and directly affects
which persona "wins."

**Takeaway:** Empathetic persona wins consistently across both  graders (8.7 with Haiku, 8.8 with Sonnet), confirming it as the most reliable choice for general-purpose support agents.

The most significant finding is the Urgent persona's collapse under Sonnet grading — empathy score dropped from 7.1 to 4.5. A direct, action-first tone that Haiku considered acceptable, Sonnet flagged as lacking emotional intelligence. This means the Urgent persona carries real risk in contexts where customers are frustrated or upset — it solves the problem but leaves thecustomer feeling unheard and disappointed.

Practical implication: persona choice should depend on context. Empathetic for emotionally charged complaints (damaged items, 
late deliveries). Urgent for transactional or time-sensitive requests (account access, quick status checks). Professional as a safe default when tone requirements are unclear.

## Tech Stack

- **Python 3**
- **Anthropic Claude API** (claude-haiku-4-5, claude sonnet)
- **python-dotenv** — environment variable management

---

## Setup

```bash
git clone https://github.com/yourusername/customer-support-evaluator
cd customer-support-evaluator
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Add your API key to a `.env` file:
```
ANTHROPIC_API_KEY=your-key-here
```

Run the full pipeline:
```bash
python dataset.py   # generate complaints
python main.py      # run all personas, grade responses
python report.py    # print summary report
```
